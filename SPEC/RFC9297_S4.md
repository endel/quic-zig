# RFC 9297 §4 — Capsule Protocol

## Status: ✅ Complete

## Implementation

**File**: `src/h3/capsule.zig`

### Capsule TLV Codec

The Capsule Protocol uses a simple Type-Length-Value framing format where
Type and Length are QUIC variable-length integers:

```
Capsule Type (varint) | Capsule Length (varint) | Capsule Value (bytes)
```

### API

| Function | Description |
|----------|-------------|
| `parse(data)` | Parse one capsule from byte buffer → `{capsule, consumed}` |
| `write(writer, type, value)` | Write a generic capsule |
| `writeDatagram(writer, payload)` | Write a DATAGRAM capsule (type 0x00) |
| `isReservedCapsuleType(type)` | Check reserved pattern `0x29*N + 0x17` |
| `CapsuleIterator` | Sequential capsule parser for stream processing |

### Capsule Types

| Type | Value | Status |
|------|-------|--------|
| DATAGRAM | 0x00 | ✅ Implemented |
| Reserved | 0x29*N + 0x17 | ✅ Detection implemented |

### Design Notes

- Unknown capsule types are returned as-is by the parser (RFC 9297 requires
  receivers to silently skip unknown types).
- `CapsuleIterator` handles partial trailing data gracefully by returning `null`
  when a complete capsule cannot be parsed.
- Uses the same QUIC varint encoding as H3 frames (`packet.readVarInt`/`writeVarInt`).

### Caveats

- The `Capsule-Protocol` HTTP header field (boolean structured field, RFC 9297 §3.2)
  is not explicitly handled — HTTP header parsing is outside the scope of the H3 framing layer.
- Capsule protocol integration into H3 CONNECT streams (reading/writing capsules
  on the data stream after Extended CONNECT succeeds) is available via the codec
  but not automatically wired into `H3Connection.poll()`. Applications (e.g., proxy
  implementations) call `capsule.parse()` / `capsule.write()` directly on stream data.

### Test Coverage

- DATAGRAM capsule write + parse round-trip
- Empty DATAGRAM capsule
- Generic (custom type) capsule write + parse
- Buffer-too-short: empty input, truncated value, type-only
- Reserved type detection (0x17, 0x40, 0x69, non-reserved values)
- CapsuleIterator: multiple capsules, partial trailing data
- Large payload (8000 bytes)
